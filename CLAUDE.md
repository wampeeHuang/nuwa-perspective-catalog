# 女娲蒸馏目录 · 项目宪法

## 红线

- **禁止新增分类。** 12 个分类是策展上限，不因数据变化而扩展。
- **禁止在前端暴露未经归并的分类标签。** 所有 `category` 字段必须从允许列表中取值。
- **禁止跳过 data.js 验证。** 新增人物后必须跑 `node _scripts/gen_dl.js` 重新生成下载映射。

## 分类体系

12 个策展分类，不可增删改：

| 分类 | 说明 |
|------|------|
| 人工智能 | AI/ML 技术与产品 |
| 科学/技术 | 自然/形式科学、工程、技术哲学（系统思维、控制论归此） |
| 商业/经济 | 投资、创业、管理、产品 |
| 营销/传播 | 广告、增长、品牌 |
| 写作/内容 | 写作、内容创作、自媒体（互联网/内容创作归此） |
| 影视/创作 | 电影、动画、视频创作 |
| 设计/艺术 | 交互设计、建筑、视觉艺术（建筑与设计归此） |
| 社会/政治 | 社会学、政治、性别、全球发展（全球发展归此） |
| 教育/学习 | 教育、学习方法 |
| 哲学/思想 | 哲学、认识论、媒介哲学、技术哲学、风险哲学（科技哲学、风险与不确定性归此） |
| 心理/认知 | 心理学、认知科学、行为科学 |
| 文学/人文 | 文学、神话学、文化批评（神话与叙事归此） |

## 新增人物 checklist

1. 在 `_data/data.js` 中添加条目，`category` 必须从上表 12 个值中选取
2. 在 `_scripts/gen_dl.js` 的 `EXTRA` 中添加搜索别名
3. 运行 `node _scripts/gen_dl.js` 重新生成 `api/dl.js`
4. 将 SKILL.md 复制到 `skills/` 目录
5. `git add` 以上所有变更文件

## 技术约定

- `index.html` 是单文件纯静态站点，不依赖构建工具
- `_data/data.js` 以 `window.__CHARACTERS__ = [...]` 格式暴露数据
- CATEGORIES 在 `index.html` 中硬编码，不从数据推导
- 分类导航按钮由 `buildCatNav()` 动态生成，count badge 自动计算
- Vercel 自动部署，push 到 master 即上线
- `_runtime/` 放临时文件，不进 git

## 选择架构原则

- **Hick's Law 上限**: 12 个分类是认知阈值，超过时决策质量显著下降
- **归并优先于新增**: 新标签先在现有 12 分类中找归属，实在无归属时提交讨论
- **空分类保留**: 心理/认知等空分类保留，为未来内容预留位置
- **所有分类都在导航中可见**: 包括 0 位人物的分类，让用户知道"这里可以有你想要但还没蒸馏的人"
