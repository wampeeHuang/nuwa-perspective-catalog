# 人物 SKILL 蒸馏 Checklist

对照填写，完成后放入 `_data/data.js` 的 `__CHARACTERS__` 数组。

## 必填字段

- [ ] `id` — 唯一标识，格式 `perspective-英文名-小写`（例：`perspective-andrej-karpathy`）
- [ ] `name_cn` — 中文名
- [ ] `name_en` — 英文名
- [ ] `nickname` — 昵称（没有就不填）
- [ ] `category` — 分类，从现有 12 类中选或新增（新增需改 `CATEGORIES` 数组）
- [ ] `oneliner` — 一句话介绍，≤30 字
- [ ] `identity` — 身份描述，2-5 句话

## 心智模型（核心，≥2 个）

每个模型：

- [ ] `name` — 模型名称
- [ ] `oneliner` — 一句话解释
- [ ] `body` — 详细阐述，≥200 字（会截断 300 字喂给对话）

```
示例:
{ "name": "Software 2.0", "oneliner": "编程正从写代码转向写数据集", "body": "..." }
```

## 语录（≥5 条）

- [ ] 代表性发言，能体现思维方式
- [ ] 不要抄名人名言网站，要从著作/访谈/演讲中提取
- [ ] 优先选有场景感的（"在 X 场合下他说 Y"）

## 决策启发式（≥3 条）

- [ ] 判断/决策时可用的原则
- [ ] 格式：一条完整的句子，可自包含理解

## 表达 DNA

- [ ] 描述该人物的语言风格、口吻、节奏
- [ ] 写成像给演员的角色说明："你会用短句。你讨厌行话。你举例时喜欢用物理类比。"

## 价值观 + 内在张力

- [ ] `values` — 核心价值取向（≥3 个关键词/短句）
- [ ] `tensions` — 矛盾/不一致之处（≥1 个）—— 这是让人物立体的关键

## 时间线（可选）

- [ ] `timeline_highlights` — 关键年份 + 事件（≥3 个）

## 校验

- [ ] 所有字段用英文双引号，JSON 合法
- [ ] `id` 不重复
- [ ] 心智模型 body 里有实质内容，不是空壳
- [ ] 提交前在本地 `npx vercel dev -p 3090` 验证页面正常渲染
